The problem

Why data anomalies go undetected

Manual checks don't scale. By the time a broken dashboard surfaces a problem, bad data has already propagated through dependent systems.

Schema changes break models

Upstream schema changes propagate silently, leaving downstream models running on columns that no longer exist or have changed type.

Volume drops go unnoticed

A missing partition or failed ingestion job can reduce row counts significantly before anyone checks a report and notices the gap.

Freshness SLAs fail quietly

Pipelines stall without triggering alerts, and stakeholders consume stale data for hours before the delay is discovered.

Null spikes corrupt training data

A sudden rise in null values in a key column can corrupt ML training sets, producing models that degrade without an obvious cause.

The solution

A better way to detect data anomalies

DataHub's assertion engine covers every layer where data quality can degrade, with ML-powered baselines that adapt to your data's real behavior.

Smart baselines

Assertions learn what normal looks like

DataHub observes historical patterns across your tables and builds baselines for volume, freshness, and column distributions without requiring manual threshold configuration.

  • Baselines update as your data patterns evolve
  • Seasonality and trends are modeled automatically
  • No threshold tuning required to get started
Full coverage

Coverage across every anomaly type

Assertions run across freshness, volume, schema, and column-level metrics, giving your team visibility into every layer where data quality can degrade.

  • Freshness checks against configurable SLA windows
  • Volume assertions detect drops and unexpected spikes
  • Column profiling flags null rates and type drift
Scale

Monitoring rules that scale automatically

Assertion templates apply across hundreds of tables at once. As new sources are added, monitoring coverage extends without additional configuration from your team.

  • Templates propagate rules across table groups
  • New tables inherit assertions from parent datasets
  • Coverage gaps surface in the observability dashboard
Alerting

Alerts with lineage context included

Every alert includes the upstream lineage path, the assertion that fired, and the downstream assets at risk, so your team can act on the right problem first.

  • Lineage graph attached to every alert payload
  • Downstream impact scope shown at time of alert
